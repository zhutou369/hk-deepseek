### 為什麼本地部署 DeepSeek？
數據私隱安全有保障，響應速度快，環境可控方便優化。Systemd 後台守護，系統重啟自動恢復，無需手動干預，根本不用你操心！

### 準備工作：硬件與軟件需求
Ubuntu 22.04 LTS (64-bit) 系統需求：CPU (多核)；RAM (至少16GB，大模型建議32GB+，內存是硬傷啊)；GPU (推薦CUDA/ROCm顯卡，我親測8GB+ VRAM顯卡效果拔群)；儲存 (預留20-30GB)。

### 步驟一：安裝及設定 Ollama 平台
Ollama 簡化大模型本地部署。我用 `v0.1.33`，DeepSeek 支援良好。
1.  **安裝 Ollama**：終端機執行 `curl -fsSL https://ollama.com/install.sh | sh` ([Ollama 官網](https://ollama.com))。
2.  **下載 DeepSeek 模型**：例如 `deepseek-coder:7b`：`ollama pull deepseek-coder:7b` (速度依網速)。
3.  **測試模型**：`ollama run deepseek-coder:7b`，輸入指令看回應即成功！

### 步驟二：建立 DeepSeek 的 Systemd 服務檔
為 DeepSeek 設置 Systemd 服務，實現後台持續運行、開機自啟。
1.  **創建服務文件**：`sudo nano /etc/systemd/system/ollama.service`
2.  **配置內容**：貼入：`[Unit]\nDescription=Ollama Service\nAfter=network-online.target\n\n[Service]\nExecStart=/usr/local/bin/ollama serve\nUser=your_username\nGroup=your_username\nRestart=always\nRestartSec=3\n\n[Install]\nWantedBy=default.target\n`
**注意**：`your_username` 務必換為實際用戶名，確保 Ollama 權限正確。
3.  **保存退出**：`Ctrl + O` -> `Enter` -> `Ctrl + X`。

### 步驟三：啟用與管理 Systemd 服務
服務文件創建後，重載 Systemd 配置並啟動服務。
1.  **重載配置**：`sudo systemctl daemon-reload`
2.  **啟用服務 (開機自啟)**：`sudo systemctl enable ollama`
3.  **啟動服務**：`sudo systemctl start ollama`
4.  **檢查狀態**：`sudo systemctl status ollama`。顯示 `active (running)` 即運行正常！

### 常見問題與解決方案
*   **Q1: Ollama 服務啟動失敗，權限錯誤。**
    *   **A1**: 檢查 `/etc/systemd/system/ollama.service` 的 `User`/`Group` 是否正確，確認 `/usr/local/bin/ollama` 有執行權限 (`sudo chmod +x /usr/local/bin/ollama`)。
*   **Q2: 模型響應慢或無反應。**
    *   **A2**: 多半內存/顯卡不足。確認資源充足，驅動正確。可嘗試小規模 DeepSeek 模型 (如 3B 版)，雲主機/虛擬機記得分配合適資源。
*   **Q3: 如何從外部網絡訪問？**
    *   **A3**: Ollama 默認監聽 `127.0.0.1:11434`。需修改 Ollama 啟動 (如 `OLLAMA_HOST=0.0.0.0`)，開放防火牆 11434 端口。注意網絡安全。
*   **Q4: 如何更新 Ollama 或 DeepSeek 模型？**
    *   **A4**: 更新 Ollama：重跑安裝命令 `curl -fsSL https://ollama.com/install.sh | sh`。更新模型：`ollama pull deepseek-coder:7b`。
