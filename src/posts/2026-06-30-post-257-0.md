#### 第三步：設計與實作RAG工作流

Retrieval Augmented Generation (RAG) 係將 LLM 模型應用到特定業務場景嘅強力工具。透過 RAG，你可以讓 DeepSeek 根據你公司嘅專有知識庫來生成答案，而唔係單純依靠模型訓練時嘅通用知識。呢個對於好多 SaaS 產品嚟講至關重要，例如我哋嘅客服機械人需要回答基於最新產品手冊嘅問題。實作 RAG 嘅基本流程包括：

1.  **數據準備與向量化**：將你嘅知識文檔（例如產品說明、FAQ、博客文章）切分成細小嘅塊，然後利用 Embedding 模型將佢哋轉換成向量，儲存到向量數據庫（例如 ChromaDB 或 Pinecone）。
2.  **用戶查詢處理**：當用戶提出問題時，將用戶查詢也轉換成向量，然後喺向量數據庫中搜尋最相關嘅文檔塊。
3.  **提示詞工程**：將搜尋到嘅相關文檔同埋用戶嘅原始查詢組合成一個精心設計嘅提示詞，然後發送俾 DeepSeek 模型。呢個步驟直接影響模型生成答案嘅品質。如果你想深入了解提示詞嘅設計，可以參考我哋另一篇關於 <a href="/posts/deepseek-prompt-basics/">DeepSeek 提示詞基本教學</a>。
4.  **模型響應**：DeepSeek 根據帶有上下文嘅提示詞生成精確嘅答案，然後返回俾用戶。

**Q2：DeepSeek 生成嘅答案唔準確或者唔夠自然？**

A2：呢個通常係提示詞工程嘅問題。你可以嘗試以下方法：

*   **few-shot learning**：喺提示詞中提供少量嘅例子，引導模型生成更符合期望嘅結果。
