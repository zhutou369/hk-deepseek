## 第二步：下載並加載 DeepSeek-Coder-V2 模型

環境都OK了，下一步就是把DeepSeek模型『請』到咱們電腦里，準備開跑！

1.  **Ollama幫忙下載模型:** 打開命令提示符，輸入這行命令就能下載DeepSeek-Coder-V2的7B Instruct版本：
    ```bash
    ollama run deepseek-coder-v2
    ```
    Ollama會自動檢測你本地有沒有這個模型。沒有的話，它就會自己去模型庫裡拖下來。提醒一句，這模型文件不小（7B的大概4GB），所以網速得給力點，硬盤空間也得留夠哈。
2.  **第一次運行和模型加載:** 模型下載完畢，Ollama會自動加載並進入互動模式。看到 `>>>` 這個提示符，就說明你能直接跟DeepSeek聊天啦！第一次加載模型會稍微慢點，畢竟是在CPU模式下，要把模型那些『腦袋』都裝進內存裡，需要時間。

*   **它思考了多久:** 大約 25-30 秒
*   **生成內容（代碼）:**

        """
        Calculates the sum of all even numbers in a list.

        Args:
            numbers: A list of integers.

        Returns:
            The sum of even numbers.
        """
        total = 0
        for num in numbers:
            if num % 2 == 0:
                total += num
        return total

    print(sum_even_numbers(my_list)) # Output: 30

## 常見問題 Q&A

### Q1: Ollama裝不上或者啟動不了，怎麼破？

**A:** 多半是端口被佔了，或者權限不夠。你得查查有沒有別的程序佔著11434端口（Ollama默認用的就是這個）。Windows用戶可以試試右鍵『管理員身份運行』安裝程序。Linux的話，看看是不是Docker或者其他虛擬化服務在搗亂。

### Q2: DeepSeek模型反應超級慢，有啥辦法能提速嗎？

**A:** CPU模式下慢是正常的，沒辦法，物理限制擺在那兒。但你可以試試下面幾招：

*   **加內存！加內存！加內存！** 重要的事情說三遍！內存越大，模型跑起來越順滑。
*   **換個小點的模型:** DeepSeek-Coder-V2其實也有1.3B這種迷你版。雖然能力會打點折扣，但跑起來會快N多倍。
*   **關掉不用的程序:** 讓Ollama和DeepSeek獨享系統資源，別讓它們跟別的軟件搶內存和CPU。
*   **升級CPU:** 有條件的話，換個更猛的CPU確實有幫助，但效果可能不如加內存明顯。

### Q3: 內存佔用太高，電腦卡到爆炸甚至死機了，怎麼辦？

**A:** 這基本就是你內存不夠用了，帶不動選的模型。趕緊換個內存需求小的模型試試，或者直接攢錢升級電腦內存。在Ollama裡面，你可以用 `ollama rm deepseek-coder-v2` 把當前模型刪了，然後再用 `ollama run deepseek-coder-v2:1.3b`（如果這個小模型存在的話）來跑。
